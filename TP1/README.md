# TP1 — MongoDB Security Lab

## Objectif
Installer MongoDB avec Docker et analyser la sécurité.

## Étapes réalisées
- Installation Docker
- Lancement MongoDB container
- Connexion via mongosh
- Insert / Read / Update / Delete

## Observations sécurité
- Port 27017 exposé sur 0.0.0.0
- Authentification désactivée
- Accès libre aux données

## Conclusion
La base MongoDB est vulnérable dans cette configuration.
